## 1. V1.1 概述

### 定位

V1.1 是 V1.0 发布后的首次迭代，聚焦三个方向：

1. **网络能力**：从纯本地工具升级为支持在线导入（ClawHub / GitHub）+ SSH 远程读写
2. **编辑增强**：修改追踪、Diff 对比、拖拽排序、跨文件校验、子节点删除
3. **体验打磨**：触发条件数据真实性、跨文件类型一致性、批量操作、向导精简

### V1.1 核心价值（相对 V1.0 新增）

| # | 能力 | V1.0 | V1.1 |
|---|------|------|------|
| 1 | 数据来源 | 仅本地 `~/.openclaw/` | + ClawHub 搜索下载 + GitHub URL 导入 + SSH 远程服务器 |
| 2 | 修改感知 | 单一 `dirty` 布尔 | 区域级变更检测 + 行级 Diff 对比 |
| 3 | 批量操作 | 逐个手动 | 批量验证 + 批量导出 |
| 4 | 文档编辑 | doc/exec 只读 | exec 区块可编辑（镜像 doc 模式） |
| 5 | 校验深度 | 单文件字段级 | + 5 条跨文件引用校验 |
| 6 | 列表操作 | 增删改 | + 拖拽排序（sources/topics） |
| 7 | 子节点管理 | 仅查看 | + config/extra 文件删除 |

### 产品形态（V1.1 更新）

- **Tauri 桌面应用**：V1.0 全部能力 + SSH 连接/自动重连 + 本地文件写入
- **纯 Web 版**：V1.0 全部能力 + ClawHub/GitHub 在线导入预览（不可保存到本地）
- **主题**：仍为纯暗色模式（V1.1-主题推迟到后续版本）
- **语言**：中文/英文，i18n 从 ~330 行扩展到 ~454 行

## 2. 新增功能模块

### 2.1 在线导入（ClawHub + GitHub）

#### 功能描述

通过 Import Dialog 从两个在线来源导入 Skill 到本地工作区。

#### 交互细节

**Import Dialog**（双标签页）

- 入口：Navigator 面板 `+` 按钮 → 下拉菜单 → "在线导入"
- 两个 Tab：ClawHub / GitHub

**ClawHub 标签页**

- 搜索框：输入关键词 → 调用 `clawhub.ai/api/search?q=...` → 单列列表展示结果
- 每行：displayName + summary + slug + updatedAt + 内联下载按钮
- 下载：`/api/v1/download?slug={slug}` → `importSkillBundle()` 写入本地 → 解析加入列表
- 支持可选 API Token（用于限流绕过，持久化到 `~/.openclaw/skillforge-config.json`）

**GitHub 标签页**

- URL 输入：粘贴公开 GitHub 仓库/目录 URL（如 `github.com/anthropics/anthropic-cookbook/tree/main/skills/web-search`）
- URL 解析：自动提取 owner/repo/branch/path
- 目录预览：通过 GitHub Contents API 递归获取目录结构，展示文件列表 + 大小
- 导入：递归下载全部文件 → `importSkillBundle()` 写入本地
- 支持可选 GitHub Token（提高 API 限额，持久化）

#### 技术实现

| 关键点 | 实现 |
|--------|------|
| HTTP 基础设施 | `tauri-plugin-http`（Rust 侧）+ JS 绑定，Tauri 环境用 Tauri HTTP；非 Tauri 降级为浏览器原生 `fetch` |
| 环境检测 | `getFetch()` 检查 `window.__TAURI__`，动态选择 HTTP 实现 |
| ClawHub API | `clawhub-api.ts`：`searchSkills()` / `getSkillMeta()` / `downloadSkill()` |
| GitHub API | `github-api.ts`：URL 解析 + Contents API + 递归目录下载 |
| 导入编排 | `skill-importer.ts`：下载 → `importSkillBundle()` 写本地磁盘 → `parseSkillMd()` 解析 → 返回 `ParsedSkill`（带 `SkillOrigin`） |
| URL 白名单 | `capabilities/default.json`：`*.openclaw.ai` / `api.github.com` / `raw.githubusercontent.com` |
| 配置持久化 | `app-config.ts`：SSH 配置 + GitHub Token 存储到 `~/.openclaw/skillforge-config.json` |

#### 涉及文件

- `src/components/import-dialog/import-dialog.tsx`（Dialog 主体 + Tab 切换）
- `src/components/import-dialog/clawhub-tab.tsx`（ClawHub 搜索/下载 UI）
- `src/components/import-dialog/github-tab.tsx`（GitHub URL 解析/预览/下载 UI）
- `src/lib/clawhub-api.ts`（ClawHub API 封装）
- `src/lib/github-api.ts`（GitHub API 封装）
- `src/lib/skill-importer.ts`（导入编排层）
- `src/lib/app-config.ts`（应用配置持久化）
- `src/lib/tauri-fs.ts`（新增 `importSkillBundle()`）

### 2.2 SSH 远程读写

#### 功能描述

通过 SSH/SFTP 连接远程 OpenClaw 服务器，浏览、下载、编辑并回写远程 Skill 文件。

#### 交互细节

**连接**

- 入口：Navigator 面板 Server 图标按钮
- SSH Connect Dialog：主机 / 端口 / 用户名 / 私钥路径 / 远程 Skill 目录
- 连接状态：图标旁绿色圆点指示已连接
- 自动重连：应用启动时自动加载已保存的 SSH 配置并尝试连接，失败静默不打扰

**远程浏览**

- 连接成功后弹出 Remote Skill List 面板
- 展示远程服务器 `/root/.openclaw/workspace/skills/` 下的 Skill 目录列表
- 每个 Skill 一行，可点击"加载到本地"

**加载到本地**

- 通过 SFTP 递归读取 Skill 目录下全部文件
- 写入本地 `~/.openclaw/workspace/skills/` 目录
- 解析后加入工作区列表，带 Cloud 图标标识 SSH 来源

**保存回写**

- SSH origin 的 Skill 保存时自动通过 SFTP 回写远程服务器
- 未连接时提示"已保存到本地缓存"

**配置持久化**

- 连接配置保存到 `~/.openclaw/skillforge-config.json`
- 下次打开 Dialog 自动加载上次配置

#### 技术实现

| 关键点 | 实现 |
|--------|------|
| Rust SSH 模块 | `ssh.rs`（~280 行）：基于 `russh` / `russh-sftp` / `russh-keys` |
| SSH Manager | `SshManager` 结构体：管理连接状态、SFTP 会话 |
| Tauri Commands | 6 个命令：`ssh_connect` / `ssh_disconnect` / `ssh_status` / `ssh_list_skills` / `ssh_read_file` / `ssh_write_file` |
| 前端封装 | `remote-fs.ts`：对 6 个 Tauri commands 的 `invoke` 封装 |
| Origin 追踪 | `SkillOrigin` 类型：`local` / `clawhub` / `github` / `ssh`，存入 `ParsedSkill` |
| 自动重连 | `navigator-panel.tsx` mount `useEffect`：加载配置 → 尝试连接 → 失败静默 |

#### 涉及文件

- `src-tauri/src/ssh.rs`（Rust SSH/SFTP 模块）
- `src-tauri/Cargo.toml`（新增 russh/russh-sftp/russh-keys 依赖）
- `src/lib/remote-fs.ts`（前端 SSH API 封装）
- `src/components/ssh-panel/ssh-connect-dialog.tsx`（连接配置 UI）
- `src/components/ssh-panel/remote-skill-list.tsx`（远程浏览 UI）
- `src/components/workspace/navigator-panel.tsx`（Server 按钮 + 自动重连）
- `src/components/workspace/inspector-panel.tsx`（`handleSaveAll` SSH 回写逻辑）

### 2.3 配置 Diff 对比

#### 功能描述

编辑态下查看当前修改与原始数据的行级 Diff，帮助用户在保存前确认变更范围。

#### 交互细节

**入口**（两处）

1. Inspector 面板标题栏的 amber "N 处修改" badge（可点击）
2. 工具栏新增 ↔ Diff 图标按钮

**Diff Viewer Dialog**

- 左栏：变更文件列表，每个文件显示状态图标（修改/新增/删除）+ 增删行统计
- 右栏：Unified diff 渲染，行号 + 加减标记 + 绿色（新增行）/ 红色（删除行）着色
- 区域覆盖：frontmatter / body / configFiles / extraFiles 各自独立对比

#### 技术实现

| 关键点 | 实现 |
|--------|------|
| Diff 算法 | `diff` npm 包（jsdiff）的 `diffLines()` 行级对比 |
| 序列化对比 | `skill-differ.ts`：将 `originalSnapshot` vs 当前 `editState` 各区域序列化为文本 → `diffLines()` → `AreaDiff[]` |
| 数据基准 | 依赖 V1.1 修改追踪的 `originalSnapshot`（`structuredClone` 编辑初始数据） |
| UI 组件 | `diff-viewer.tsx`：Dialog + 左栏文件列表 + 右栏 unified diff 渲染 |

#### 涉及文件

- `src/lib/skill-differ.ts`（Diff 逻辑）
- `src/components/workspace/diff-viewer.tsx`（Diff UI）
- `src/components/workspace/inspector-panel.tsx`（badge 点击 + ↔ 按钮）

### 2.4 批量操作

#### 功能描述

对工作区全部已加载 Skill 执行批量验证和批量导出。

#### 交互细节

**入口**

- Navigator 面板标题栏的 Layers 按钮 → 下拉菜单 → "批量验证" / "批量导出"

**批量验证**

- Dialog 表格展示每个 Skill 的验证状态：名称 + error 数 + warning 数 + info 数
- 底部汇总统计：总 Skill 数、通过数、有问题数

**批量导出**

- Dialog 列表展示全部 Skill，每个 Skill 显示包含的文件（SKILL.md + configFiles + extraFiles）
- "全部导出"按钮 → 将全部 Skill 序列化为 JSON manifest → 浏览器下载

#### 技术实现

| 关键点 | 实现 |
|--------|------|
| 验证引擎 | 复用 `validateSkill()` 对全部 skills 循环执行 |
| 导出序列化 | 将每个 Skill 的 SKILL.md + configFiles + extraFiles 打包为 JSON manifest |
| UI 组件 | `batch-dialog.tsx`（~190 行）：验证表格 + 导出列表 + 下载按钮 |

#### 涉及文件

- `src/components/workspace/batch-dialog.tsx`
- `src/components/workspace/navigator-panel.tsx`（批量按钮 + 下拉菜单）

## 3. 增强功能

### 3.1 修改追踪增强

**V1.0**：单一 `dirty: boolean`，仅知道"有没有改"。

**V1.1 增强**：

- `SkillEditState` 新增 `originalSnapshot`：编辑初始化时 `structuredClone` 原始数据
- `computeChanges(editState)` 工具函数：对比当前 vs 原始，返回 `{ totalCount, areas[] }`
- Navigator 树节点：amber 数字 badge（替代小圆点），显示修改区域数量
- Inspector 面板："N 处修改" badge（替代"已修改"），hover 显示具体区域名称
- 为 V1.1-Diff 提供数据基准

**涉及文件**：`workspace.ts` / `use-workspace.ts` / `navigator-panel.tsx` / `inspector-panel.tsx`

### 3.2 跨文件校验

**V1.0**：仅单文件字段级校验（名称格式、必填字段等）。

**V1.1 新增 5 条 warning 级别跨文件规则**：

| 规则 | 级别 | 条件 |
|------|------|------|
| 文件读引用不存在 | warning | `files.read[]` 引用的路径不在 Skill 文件列表中 |
| 文件写路径为空 | warning | `files.write[]` 含空字符串 |
| 数据源引用无效主题 | warning | sources.json 引用了不存在的 topic ID |
| 环境变量未被引用 | warning | env 变量名未在任何代码文件中出现 |
| 脚本引用不存在 | warning | Markdown body 中引用的脚本文件不在 Skill 目录中 |

**实现**：`mergeSkillForValidation()` 扩展支持 `extraFiles`，校验函数同时接收配置文件和附加文件数据。

**涉及文件**：`skill-validator.ts` / `editor-panel.tsx`

## 4. 修复与优化

### 4.1 触发条件虚假默认值修复

**问题**：`TriggerDisplay`（编辑面板的触发条件区块）硬编码 9 个字段行，无论 frontmatter 中有没有值都全显示。大多数 Skill 只配了 `user-invocable: true`，但 UI 显示 8 行 "—"。

**修复**：
- TriggerDisplay 只渲染 frontmatter 中实际存在的字段
- 无配置时显示"未配置触发条件"空态
- `saveTrigger` 不再无条件写回默认值

**涉及文件**：`editor-panel.tsx` / i18n

---

### 4.2 跨文件类型展示一致性

**问题**：config 文件的编辑/预览面板与 SKILL.md 风格不一致（不同容器组件、不同 CSS 体系）。

**修复**（3 个 Phase）：

| Phase | 内容 |
|-------|------|
| A 编辑面板统一 | SourcesEditor/TopicsEditor 重构为 SectionBlock 包裹 + 展示/编辑切换 + draft 模式 |
| B 预览面板结构化 | config 预览从 JsonPreview 升级为 SourcesPreview/TopicsPreview/SchemaJsonPreview + sectionId 桥线 |
| C 清理 + 嵌套 JSON 修复 | 删除 5 个废弃文件（-632 行）；嵌套对象/数组编辑保存修复 |

**涉及文件**：`sources-editor.tsx` / `topics-editor.tsx` / `inspector-panel.tsx` / `editor-panel.tsx`

---

### 4.3 schema.json 可编辑

**V1.0**：schema.json 路由到只读 `SchemaViewer`。

**V1.1**：新增 `SchemaRawEditor`——展示态保持 SchemaViewer 格式化表格，编辑态切换为 textarea + JSON parse 校验。

**涉及文件**：`editor-panel.tsx`

---

### 4.4 Markdown 统一解析架构

**V1.0 限制**：SKILL.md 用正则解析器，附加文件用 remark AST，两套并存。

**V1.1 统一**（5 个批次）：

| 批次 | 内容 |
|------|------|
| B1 底座 | `ParsedSkill` 类型扩展 `parsedBody: ParsedDocument`；`skill-parser.ts` 调用 `parseDocument()` |
| B2 编辑面板 | doc/exec 区块替换为 `FragmentBlock` 统一渲染 |

### 5.5 Rust 后端扩展

**V1.0**：仅 `lib.rs`（fs/dialog/log 插件注册）。

**V1.1 新增 `ssh.rs`**：

```
SshManager (Arc<Mutex<Option<SshSession>>>)
├── ssh_connect(host, port, username, key_path, remote_path)
│   → russh_keys 解析密钥 → russh 建立连接 → russh_sftp 开启 SFTP 会话
├── ssh_disconnect()
│   → 关闭 SFTP + SSH 连接
├── ssh_status()
│   → 返回 { connected, host?, remote_path? }
├── ssh_list_skills()
│   → SFTP readdir → 过滤目录 → 返回目录名列表
├── ssh_read_file(path)
│   → SFTP open + read → 返回文件内容 String
└── ssh_write_file(path, content)
    → SFTP create + write → 写入文件
```

`lib.rs` 更新：注册 `tauri-plugin-http` + 6 个 SSH commands。
