# V1-UX 用户反馈修正 执行记录

**日期**: 2026-04-08
**执行模式**: Opus 直接编写（修正量小、逐项调整）

## 调研记录

纯修正任务，对照用户反馈逐项调整 V1-UX 的产出，无需调研。

## 背景

V1-UX 发布前优化（[V1-UX 优化会话](b7fdaa2f-97a9-4182-a0ac-9ee985ff2c70)）完成后，用户逐项审查提出 4 类问题：

1. 图标大小统一方向错误（缩小了不该缩小的）
2. 区块分隔线太弱，没有区分感
3. 标题动态切换过度设计
4. 架构条动态隐藏不合理

## 批次执行

### 修正 1：图标大小方案

**问题**：V1-UX 将 ShieldCheck 从 h-5(20px) 缩小为 size-4(16px)，与校验图标"统一"了。但用户认为方向错误——应该把校验图标放大到匹配 ShieldCheck，而不是把 ShieldCheck 缩小。桥线区链接图标也仍偏小。

**方案**：建立三档图标体系
- S 12px (`size-3`)：tag 删除 X、行内辅助
- M 14px (`size-3.5`)：面板标题、文件图标、操作按钮 — 大多数图标
- L 20px (`size-5`)：校验/状态图标（ShieldCheck、AlertCircle 等）

**改动**：
- `editor-panel.tsx`：ShieldCheck `size-4` → `size-5`
- `validation-panel.tsx`：CheckCircle2/AlertCircle/AlertTriangle/Info 全部 `size-4` → `size-5`
- `bridge-connector.tsx`：链接 SVG `h-5 w-5` → `h-[22px] w-[22px]`，容器 28×28 → 32×32

### 修正 2：区块分隔线增强

**问题**：`border-bottom: 1px solid rgba(255,255,255, 0.04)` 在暗色背景上几乎不可见。区块间距 4px 也太小。

**参考**：Demo 05 用 `margin-bottom: 8px` + 无 border，纯靠间距分隔。

**改动**：
- `index.css`：`[data-bridge-section]` margin-bottom 4px → 8px，border-bottom opacity 0.04 → 0.08
- 两侧面板自动同步（共用选择器）

### 修正 3：标题逻辑简化

**问题**：V1-UX 将 extra-file 页面标题改为"文件查看"、config-file 改为"配置编辑"。但用户指出"可视化编辑"和"源码预览"两个标题对所有页面都适用，不需要改。

**改动**：
- `editor-panel.tsx`：去掉 config-file/extra-file 的特殊标题分支，非概览页统一显示"可视化编辑"

### 修正 4：架构条行为调整

**问题**：V1-UX 让空层级不显示，但用户认为：
1. 所有区块无论是否有数据都会在编辑/预览区展示，层级按钮不应隐藏
2. 应始终显示 6 个层级，无数据的置灰/弱化表示
3. 选中层级后应滚动到对应区块

**方案讨论**：提供 A(半透明)/B(去饱和)/C(虚线轮廓) 三种空层级表示方案，用户选择 C。

**改动**：
- `architecture-bar.tsx`：
  - 始终渲染全部 `BRIDGE_LAYERS`，不再 filter
  - `hasLayerContent` 保留用于判断是否为空层（添加 `.empty` class）
  - 所有层级可点击，点击后触发 `scrollBothToSection` 滚动到首个区块
- `index.css`：新增 `.arch-chip.empty`（transparent 底色 + `1px dashed` 边框 + `opacity: 0.4`）
