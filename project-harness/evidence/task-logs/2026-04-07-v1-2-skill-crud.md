# V1-2 技能 CRUD 入口 执行记录

**日期**: 2026-04-07

## 调研记录

豁免调研。理由：shadcn/ui DropdownMenu 和 AlertDialog 是已有技术栈内的重复模式（shadcn v4 + @base-ui/react）；FileReader API 是标准 Web API；workspace reducer 模式已在项目中建立。

## 批次执行

### 步骤 1：安装 shadcn/ui 组件
- 操作：`npx shadcn@latest add dropdown-menu alert-dialog`
- 结果：生成 `src/components/ui/dropdown-menu.tsx` 和 `alert-dialog.tsx`

### 步骤 2：workspace 状态扩展
- 操作：
  - `workspace.ts`：新增 `ADD_SKILL` / `REMOVE_SKILL` action 类型
  - `use-workspace.ts`：reducer 新增两个 case + context 暴露 `addSkill`/`removeSkill`
- 结果：`ADD_SKILL` 自动跳过重复 ID 并选中新技能；`REMOVE_SKILL` 清理 editStates 并回退选中项

### 步骤 3：NavigatorPanel 重写
- 操作：
  - `+` 按钮改为 `DropdownMenu`（@base-ui/react Menu primitive）
  - 3 个菜单项：创建新技能（复用 SkillWizard）/ 导入 SKILL.md（file input）/ 粘贴内容（Dialog + textarea）
  - `deduplicateId()` 处理重名（追加 `-N` 后缀）
  - `importSkillFromContent()` 统一解析入口（file upload 和 paste 共用）
  - 上传失败显示 error banner（可关闭）
- 结果：三个入口均可正常触发各自的交互流程

### 步骤 4：删除功能
- 操作：
  - `SkillTreeBlock` 新增 `onDelete` prop + hover 显示 `Trash2` 图标
  - `AlertDialog` 确认弹窗（"移除技能 — 不会删除源文件"）
  - 确认后调用 `removeSkill()`
- 结果：hover 显示删除按钮 → 点击弹出确认 → 确认后从列表移除
