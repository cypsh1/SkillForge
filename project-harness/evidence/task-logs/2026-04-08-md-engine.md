## 调研记录

### 搜索内容
- 搜索 1："unified remark mdast markdown AST parser JavaScript 2026" → remark-parse 是 AST 导向的 Markdown 处理器，支持 parse + stringify 双向操作
- 搜索 2："markdown visual block editor React component library 2026" → MDXEditor 是 WYSIWYG 方向（太重），确认自建 fragment 渲染层更合适
- 搜索 3："remark-parse vs markdown-it vs marked comparison 2026" → marked 最快但无 AST，markdown-it 扁平 token，remark-parse 树形 AST + 双向

### 调研结论
- 选定 remark-parse (unified 生态)，对比过 markdown-it（扁平 token + 无反序列化）和 marked（无 AST + 单向）
- 不引入 WYSIWYG 编辑器（如 MDXEditor），因为与现有 bridge-section 组件体系冲突，且 V1.0 不需要
