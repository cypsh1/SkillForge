# V1.1-STYLE-UNIFY：跨文件类型展示与编辑一致性

**日期**：2026-04-10
**状态**：✅ 完成

## 背景

用户排查发现 config 文件（sources.json / topics.json / schema.json）的编辑和预览面板与 SKILL.md 风格不一致——打开 SKILL.md 是紧凑、有联动的专业界面，切到 config 文件变成另一种卡片布局，不像同一个产品。

## 执行记录

### Phase A：编辑面板容器统一

**改动文件**：sources-editor.tsx, topics-editor.tsx, editor-panel.tsx

- SourcesEditor：移除 shadcn Card/Badge/Switch/Label/Input → SectionBlock 包裹 + 展示态（`.fr` 紧凑行）/ 编辑态（`.ecard` + `.ef-row` + `.ftg` 控件）+ draft 模式
- TopicsEditor：同上模式重构
- ConfigFileEditor：错误态从 `Card > CardContent` 改为 `<p>` 简文本；删除 Card 导入
- 配色：config 文件统一使用 `#06b6d4`（cyan）

### Phase B：预览面板结构化 + 桥线联动

**改动文件**：inspector-panel.tsx, sources-editor.tsx, topics-editor.tsx, editor-panel.tsx

- 新增 ConfigFilePreview 路由组件：sources → SourcesPreview, topics → TopicsPreview, schema → SchemaJsonPreview
- 替换 JsonPreview（原始 JSON 文本）为结构化行预览
- 添加 sectionId（cfg-sources, cfg-topics, cfg-schema）到编辑和预览两侧的 SectionBlock，启用桥线同步
- 删除 JsonPreview 导入

### Phase C：清理 + 嵌套 JSON 修复

**删除文件**（5 个，-632 行）：
- `src/pages/skill-detail.tsx` — 废弃页面，无导入
- `src/pages/skill-list.tsx` — 废弃页面，无导入
- `src/components/config-editor/config-editor.tsx` — 被 workspace 编辑器取代
- `src/components/config-editor/json-preview.tsx` — 被 inspector-panel 内部组件取代
- `src/components/layout/app-sidebar.tsx` — 从未导入
- 删除空 `src/pages/` 目录

**修复文件**：extra-file-editors.tsx

- JsonFileEditor 嵌套对象/数组修复：
  - 展示态：`formatJsonValue` 对 object/array 使用 `JSON.stringify` 代替 `String()`（消除 `[object Object]`）
  - 编辑态：嵌套值使用 textarea + JSON 格式化代替 input
  - 保存时：`handleField` 对嵌套原始类型尝试 `JSON.parse` 恢复层级
