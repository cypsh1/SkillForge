### Batch 4：实体规则全面动态化
- 操作：
  1. 重写 `bridge-relations.ts`：删除硬编码 BRIDGE_RELATIONS 常量 → 改为 setRelationStore/getRelationStore 动态 store + buildBridgeRelations 启发式生成器
  2. `use-panel-sync.ts`：usePanelSync 接收 skill/fm 参数，useMemo 中调用 buildBridgeRelations 并更新 store
  3. `App.tsx`：WorkspaceShell 中计算 selectedSkill/editFm 传入 usePanelSync
  4. `relation-bar.tsx`：从 BRIDGE_RELATIONS 常量改为 getRelationStore() 读取
  5. 修复 extractScriptNames：parser 将每个 heading 作为独立 section，需遍历 script parent 下的所有子 section（level > parentLevel）
  6. 修复类型：requires/optionalBins 在 metadata.openclaw 下，需通过 getOpenclawMetadata 访问
- 结果：tech-news-digest 动态生成 17 个实体关系（8 个 env var + 7 个 optionalBins + python3 + run-pipeline + generate-pdf），关系气泡正常显示
- 关键 Bug：初版 extractScriptNames 只找到 7 个 optionalBins 关系，env vars 全部缺失。原因是 section filter `/脚本|script|pipeline/i` 无法匹配 `fetch-rss.py - RSS Feed Fetcher` 等子 section 标题。修复为 parent-tracking 模式后正常。
