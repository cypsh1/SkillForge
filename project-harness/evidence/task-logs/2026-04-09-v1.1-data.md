## 批次执行

### 步骤 1：S1 rsync 全量同步

- **操作**：`rsync -avz` 从 `openclaw:/root/.openclaw/workspace/skills/` 到 `src/data/test-skills/`，排除 `__pycache__/`、`*.pyc`、`*.bak*`。
- **结果**：服务器 18 个 Skill 文件同步完成；本地保留原有 `tech-news-digest-cn`（服务器无此目录），共 19 个 Skill 目录。

### 步骤 2：S2 skill-loader 重构

- **操作**：`skill-loader.ts` 使用三组 eager `import.meta.glob`；按路径解析 skill id 与相对路径，组装 `configFiles` / `extraFiles`。
- **结果**：`loadTestSkills()` / `getSkillById()` 签名不变；构建产物中 `skill-loader` chunk 增大（嵌入全部测试资源）。

### 步骤 3：S3 tauri-fs 补齐 extra files

- **操作**：`loadLocalSkills()` 在加载 config 后调用 `loadExtraFiles` → `walkTextFiles`，跳过根 `SKILL.md` 与 `config/` 子树。
- **结果**：桌面模式可加载与开发模式同类的文本 extra 文件。

### 步骤 4：S4 验证

- **操作**：`npx tsc -b --noEmit`、`npm run build`；清点本地 Skill 目录数与服务器对比。
- **结果**：tsc ✅ build ✅；本地 19 目录 vs 服务器 18（多出的为 `tech-news-digest-cn`）。
