## 任务 1：V1.1-子节点删除 — Config/Extra 文件节点删除

### 问题
Navigator 树中 config 文件和 extra 文件无法删除。只有 Skill 整体删除（`REMOVE_SKILL`），且仅从状态移除不删磁盘文件。

### 改动
- `types/workspace.ts`：`SkillEditState` 新增 `deletedConfigPaths: string[]` / `deletedExtraPaths: string[]`；`WorkspaceAction` 新增 `REMOVE_CONFIG_FILE` / `REMOVE_EXTRA_FILE`
- `hooks/use-workspace.ts`：`getOrCreateEditState` 初始化新字段；新增两个 reducer case（删 key + push deletedPaths + dirty + selection 回落）；`SAVE_SKILL` 改造（用 es.configFiles 替换 + 遍历 deletedExtraPaths 删 key + 重算 hasConfig）；context 新增 `removeConfigFile` / `removeExtraFile`
- `lib/tauri-fs.ts`：新增 `deleteSkillFile(skillPath, relativePath)` — `fs.exists()` → `fs.remove()`
- `workspace/inspector-panel.tsx`：`handleSaveAll` 在写入后遍历 `deletedConfigPaths` / `deletedExtraPaths` 调用 `deleteSkillFile`
- `workspace/navigator-panel.tsx`：`TreeNode` 新增 `onDelete` prop → hover 红色 Trash2 图标（复用 `group-hover` 模式）；新增 `deleteFileTarget` state + `AlertDialog` 确认框；`ConfigSubtree` / `ExtraFilesSubtree` 传递 `onDelete`；SKILL.md 不传（不可删除）
- i18n：新增 `deleteFile` / `deleteFileConfirm` / `delete` 中英文 key

## 任务 3：V1.1-跨文件校验 — 跨文件联动校验

### 问题
`validateSkill()` 只校验单文件 frontmatter。无法检测文件间引用断裂。

### 改动
- `lib/skill-validator.ts`：新增 `crossFileValidate(skill)` 函数，在 `validateSkill` 末尾调用。5 条 warning 级别规则：
  1. `files.read[]` 引用路径在 configFiles/extraFiles 中不存在
  2. `files.write[]` 空路径
  3. sources.json 中引用的 topic ID 在 topics.json 中不存在
  4. env 变量名未在 extra files / rawContent 中出现
  5. Markdown body 中反引号引用的 `.py/.sh/.js` 文件不在 extraFiles 中
- `workspace/editor-panel.tsx`：`mergeSkillForValidation` 新增 `extraFiles` 参数，调用处传入 `editState.extraFiles`

### 验收
- [x] tech-news-digest 概览面板校验区显示 4 个 files.write 空路径 warning
- [x] 所有跨文件问题为 warning 级别，不阻塞保存
