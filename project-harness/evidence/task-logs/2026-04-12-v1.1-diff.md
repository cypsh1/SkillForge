# V1.1-Diff — 配置 Diff 对比查看

**日期**：2026-04-12
**类型**：新功能
**状态**：✅ 完成

## 概述

编辑态新增 Diff 可视化功能。用户修改 Skill 后，点击 amber "N 处修改" badge 或 ↔ 图标按钮，打开 Dialog 弹窗查看所有变更区域的行级 diff。

## 实现细节

**依赖**：安装 `diff`（jsdiff）+ `@types/diff`，提供 `diffLines()` 行级对比算法。

**Diff 计算引擎** `src/lib/skill-differ.ts`（新建）：
- `computeSkillDiff(editState)` → `AreaDiff[]`
- 各区域序列化策略：frontmatter → YAML，configFiles → formatted JSON，body/extraFiles → 原文本
- 对比 originalSnapshot（编辑开始时的 structuredClone）vs 当前 editState
- 支持 modified / added / deleted 三种状态
- 统计每个区域的加减行数

**Diff 查看器** `src/components/workspace/diff-viewer.tsx`（新建）：
- Dialog 弹窗（max-w-4xl, max-h-80vh）
- 左栏：变更文件列表，带状态图标（FilePenLine/Plus/Trash2）+ 加减统计
- 右栏：unified diff 渲染，双列行号 + +/- 前缀 + 绿色加行/红色减行着色
- 区域标题栏显示名称 + 状态标签（已修改/新增/已删除）

**集成入口** `inspector-panel.tsx`：
- amber "N 处修改" badge 从纯展示变为可点击按钮 → 打开 DiffViewer
- 工具栏新增 ↔（ArrowLeftRight）图标按钮 → 打开 DiffViewer
- diff 数据仅在 Dialog 打开时计算（避免不必要开销）

**i18n**：`workspace.diff.*` 共 7 个 key（title/viewDiff/modified/added/deleted/frontmatter/body），zh + en 双语。

## 设计决策

- **Dialog 而非 Sheet/面板切换**：diff 审查是独立动作，需要足够空间展示跨区域变更
- **jsdiff 而非自研 LCS**：6KB gzipped、零依赖、久经考验，不值得重造轮子
- **diff 数据延迟计算**：仅在 `diffOpen=true` 时传入非空 diffs，避免每次 render 都序列化 + diff
