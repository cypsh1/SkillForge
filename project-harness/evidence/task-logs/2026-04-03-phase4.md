## 调研阶段

### 搜索方向与发现

1. **DaVinci Resolve UI 分析**（结合用户截图）

   - Tauri v2：8MB 包体 vs Electron 120MB
   - 内存 20-80MB vs 300MB
   - 内置文件系统 API

## 批次执行

### 4.1 三栏布局基础
- 安装 react-resizable-panels
- 创建 WorkspaceContext + useReducer（统一编辑状态）
- 替换旧的 Sidebar + Tab 布局为 Navigator | Editor | Inspector
- **问题**：react-resizable-panels v4 API 不同于文档示例，`PanelGroup` → `Group`，`direction` → `orientation`，无 `autoSaveId`
- **修复**：手动检查导出和类型定义，适配 API

### 4.2-4.4 三个面板（并行子 Agent）
- Navigator: 两级导航树 + 搜索 + 节点描述
- Editor: 根据选中节点动态切换编辑器
- Inspector: 预览 + 关联文件 + 导出 + 保存

### 4.5 统一编辑状态
- WorkspaceContext: skills + selection + editStates
- useReducer: SELECT / UPDATE_FRONTMATTER / UPDATE_CONFIG / RESET_EDITS
- 三栏共享同一份编辑数据

### 4.6 Tauri 桌面封装
- 安装 Rust 1.94.1 工具链
- Tauri v2 初始化 + fs/dialog 插件
- 配置文件系统权限：读写 ~/.openclaw/workspace/skills/
- 窗口 1400x900

### 4.7 本地 Skill 加载
- tauri-fs.ts: 检测 Tauri 环境 → 扫描本地 skills 目录
- 递归加载 config/ 下所有 JSON 文件
- Web 模式降级使用静态测试数据
- 检查器添加"保存"按钮

### 4.8 Skill 创建向导
- 5 步向导: 模板 → 基本信息 → 工具 → 环境变量 → 预览创建
- 4 种模板: 空白/基础工具/配置型/克隆现有
- Dialog 集成到导航面板 "+" 按钮
- 支持导出 SKILL.md 和 Tauri 本地创建
