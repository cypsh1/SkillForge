## 批次执行

### B1：底座（类型 + 解析器 + 状态管理 + 序列化）
- **操作**：
  - `types/skill.ts`：新增 `FrontmatterStatus = "valid" | "missing" | "invalid"` + `bodyDocument: ParsedDocument` + `rawFrontmatter: string | null`
  - `types/workspace.ts`：`SkillEditState` 新增 `markdownBody` / `frontmatterStatus` / `rawFrontmatter`；新增 `UPDATE_SKILL_BODY` action
  - `skill-parser.ts`：`extractFrontmatter` 重写返回 `{ frontmatter, body, status, rawFrontmatter }`；`parseSkillMd` 调用 `parseDocument(body)` 生成 `bodyDocument`
  - `use-workspace.ts`：`getOrCreateEditState` 初始化新字段；reducer 处理 `UPDATE_SKILL_BODY`
  - `skill-serializer.ts`：`serializeSkillMd` 接受 `status` 参数，`"missing"` 时跳过 frontmatter 分隔符
- **结果**：tsc ✅ build ✅

### B2：编辑面板 doc/exec 替换
- **操作**：
  - `editor-panel.tsx`：新增 `FragmentBlock` / `serializeDocument` / `ContentBlock` 导入

  - exec 区块从旧 pipeline 树 → FragmentBlock 只读渲染
  - doc 区块从 SectionsTree → 多个独立 BridgeSectionBlock + FragmentBlock（逐段编辑）
  - 删除 `SectionsTree` 导入和相关 helper
- **结果**：tsc ✅ build ✅ 浏览器验证 ✅

### B3：预览面板 + 保存链路修复
- **操作**：

  - `SectionedPreview` 的 exec/doc 改为 FragmentBlock 渲染
  - 保存链路自动包含 markdownBody
- **结果**：tsc ✅ build ✅ 浏览器验证 ✅

### B4：区块容器合并 + 非 .md 预览补齐
