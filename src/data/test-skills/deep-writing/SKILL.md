# Deep Writing System for OpenClaw

## 版本信息
- **版本**: 1.0.0
- **更新日期**: 2026-03-29
- **适用对象**: OpenClaw 深度用户
- **核心定位**: 从 0 到 1 的完整写作系统

---

## 一、系统概述

本 Skill 提供一套**通用、可扩展**的深度写作方法论，不预设写作类型（叙事/分析/理论），通过**四层诊断法**精准定位需求，结合**OpenClaw 长上下文优势**，产出具有**理论深度**和**批判思维**的高质量终稿。

### 核心创新点

1. **四层诊断法**：写作前快速诊断（带备选选项供用户选择）
2. **灵活结构策略**：根据内容选择是否使用倒金字塔
3. **理论品牌化**：合适情况下将概念升级为理论模型
4. **批判性自检**：根据内容选择深度自检清单
5. **OpenClaw 优化**：充分利用长上下文、Agent 模式、多轮迭代

---

## 二、四层诊断法（启动必做）

每次启动写作任务，必须通过以下四层诊断，**每层提供备选选项供用户选择或确认**。

### 第一层：写作类型诊断

**问题**：这是什么类型的写作？

**备选选项**（供用户选择/调整）：

| 类型 | 特征 | 典型场景 |
|------|------|----------|
| **分析型** | 剖析现象→揭示机制→推导趋势 | 行业分析、竞争研究、技术解读 |
| **论证型** | 提出论点→多维论证→回应质疑 | 观点文章、政策建议、战略提案 |
| **叙事型** | 背景铺陈→冲突展开→结局/启示 | 案例复盘、创业故事、变革记录 |
| **指导型** | 问题界定→步骤拆解→注意事项 | 操作手册、实施指南、方法论 |
| **思辨型** | 概念辨析→逻辑推演→边界探索 | 哲学讨论、范式批判、未来构想 |

**用户操作**：选择最接近的类型，或提出新的类型定义。

### 第二层：目标读者诊断

**问题**：这篇文章为谁而写？

**备选选项**（供用户选择/调整）：

| 读者类型 | 特征 | 写作策略 |
|----------|------|----------|
| **决策者** | 时间稀缺，关注结论和ROI | 倒金字塔结构， Executive Summary 前置 |
| **专业同行** | 具备背景知识，追求深度 | 理论深度，方法论详实，可复现 |
| **跨界学习者** | 有一定基础，需要上下文 | 概念解释，类比丰富，循序渐进 |
| **公众/大众** | 背景多元，关注相关性 | 故事驱动，案例具象，避免术语 |
| **实践执行者** | 需要可操作步骤 | SOP 清晰，工具推荐，避坑指南 |

**用户操作**：选择主要读者类型，或描述目标读者特征。

### 第三层：核心论点诊断

**问题**：如果只能用一句话概括，这篇文章的核心观点是什么？

**AI 辅助操作**：

如果用户已有明确论点，确认并提炼。

如果用户论点不明确，AI 基于写作类型和主题，**生成 3-5 个候选核心论点供用户选择**。

**示例格式**：

```
基于您的主题「AI Agent 对软件开发的影响」，以下是可能的候选核心论点：

1. 【范式转移型】AI Agent 正在将软件工程从「人写代码」转变为「人定义问题，AI 生成/维护代码」，这将重构开发团队的组织形态和价值链。

2. 【效率革命型】AI Agent 对软件开发的核心价值不是代码生成速度，而是将「隐性工程知识」（如调试、重构、测试）显性化、自动化，从而降低认知负荷。

3. 【风险警示型】AI Agent 的大规模应用可能导致软件开发领域出现「表层繁荣」——代码产出激增，但系统性架构能力退化，长期技术债务累积。

请选择最接近您观点的选项，或提出您的核心论点。
```

### 第四层：证据充分度诊断

**问题**：支撑核心论点的证据/数据/案例目前处于什么状态？

**备选选项**（供用户选择/调整）：

| 状态 | 特征 | 后续策略 |
|------|------|----------|
| **已有充分素材** | 数据、案例、引用已齐备 | 直接进入结构规划和写作 |
| **有部分素材** | 核心数据/案例有，但需补充 | 写作过程中标记需补充点，并行搜集 |
| **素材稀缺** | 观点先行，证据不足 | 先进行深度调研阶段，再进入写作 |
| **需要生成性研究** | 需要访谈、实验、数据收集 | 设计研究方案，写作延后 |

**用户操作**：选择当前状态，或描述具体情况。

## 三、写作流程（四层诊断后执行）

### Phase 1: 结构规划（30-60分钟）

基于四层诊断结果，确定文章结构。

**结构决策树**：

```
目标读者是否为决策者？
├── 是 → 采用倒金字塔结构
│         核心论点 → 关键证据 → 详细论证 → 背景/方法
│
└── 否 → 写作类型是什么？
          ├── 分析型/论证型 → 逻辑递进结构
          │                    问题/背景 → 分析框架 → 逐层论证 → 结论/建议
          ├── 叙事型 → 时间线/冲突驱动结构
          │           背景设定 → 冲突出现 → 高潮/转折 → 结局/启示
          ├── 指导型 → 步骤/模块结构
          │           目标/适用范围 → 步骤1 → 步骤2 → ... → 注意事项/常见问题
          └── 思辨型 → 辩证/对话结构
                       概念A的论证 → 概念B的反驳/补充 → 综合/超越 → 新的问题/方向
```

**输出**：结构大纲（含每部分字数预估、核心信息点）

### Phase 2: 初稿生成（OpenClaw 优化模式）

**OpenClaw 专用策略**：

1. **长上下文利用**：
   - 将四层诊断结果、结构大纲、参考资料一次性载入上下文
   - 使用 OpenClaw 的持久化记忆，保持写作连贯性

2. **Agent 模式分块**：
   - 复杂章节拆分为子任务，由 Agent 并行处理
   - 例如：案例分析、数据可视化描述、对比表格生成可并行

3. **多轮迭代策略**：
   - 第一轮：快速产出完整初稿（不求完美，求完整）
   - 第二轮：逐段深化（补充证据、强化论证、优化表达）
   - 第三轮：整体润色（一致性检查、节奏调整、开头结尾强化）

**输出**：完整初稿（建议字数：目标终稿字数的 120-150%）

### Phase 3: 修改完善（核心升级阶段）

此阶段应用**初稿→终稿升级方法论**。

#### Step 3.1: 诊断初稿问题

对照以下检查清单，标记初稿问题：

| 检查项 | 问题描述 | 严重程度 |
|--------|----------|----------|
| **核心论点清晰度** | 核心论点是否一句话能说清？是否前置？ | 🔴高/🟡中/🟢低 |
| **结构合理性** | 章节顺序是否逻辑顺畅？是否存在信息重复或跳跃？ | 🔴高/🟡中/🟢低 |
| **论据充分性** | 每个核心论点是否有足够证据支撑？数据/案例是否具体可信？ | 🔴高/🟡中/🟢低 |
| **理论深度** | 是否停留在现象描述？是否形成了可命名的概念/理论？ | 🔴高/🟡中/🟢低 |
| **批判性思维** | 是否考虑了反面观点？是否承认了局限性/边界条件？ | 🔴高/🟡中/🟢低 |
| **表达精炼度** | 是否存在冗余句子？专业术语使用是否一致？ | 🔴高/🟡中/🟢低 |

**输出**：问题诊断报告（优先级排序）

#### Step 3.2: 应用升级策略

根据诊断结果，选择性地应用以下升级策略：

**策略 A: 核心论点强化（倒金字塔结构）**

适用情况：核心论点不清晰、前置不足、缺乏吸引力。

操作方法：
1. 用一句话重新提炼核心论点（强制）
2. 在文章最前端增加"零章"或 Executive Summary，直接呈现核心论点+3个关键洞察
3. 后续章节作为展开论证，而非探索过程

**策略 B: 理论品牌化升级**

适用情况：概念停留在描述层面，缺乏理论深度和传播力。

操作方法：
1. 识别文章中的核心概念（如"Token消耗"）
2. 将其升级为可命名的理论模型：
   - 命名：XXX理论/模型/框架（如"Token竞争力理论"）
   - 公式化：尽可能用数学表达式或逻辑公式呈现（如"工作质量 = f(思维链, 上下文, 迭代深度)"）
   - 可视化：用文字描述图表结构
3. 在关键洞察部分，用一句话提炼理论贡献

**策略 C: 批判性深化**

适用情况：论证过于单向度，缺乏对局限性、边界条件、反面观点的考虑。

操作方法：
1. 强制增加"批判性反思"章节（独立成章或嵌入各节）
2. 必须回答以下至少3个问题：
   - 相关性≠因果性：你的核心论点的反例可能是什么？
   - 边界条件：这个理论/观点在什么情况下不适用？
   - 能力外包风险：过度依赖这个能力，人类会失去什么？
   - 边际收益递减：投入产出比何时开始下降？
   - 反面观点：最聪明的反对者会如何反驳你？
3. 对每一个批判性问题，给出诚实的回应，而非防御性辩解

**策略 D: 表达精炼与可视化**

适用情况：表达冗长、结构混乱、缺乏视觉层次。

操作方法：
1. **删减**：删除以下内容的20-30%
   - 重复论证
   - 过度铺垫（如"随着XX的发展..."类套话）
   - 已知常识（除非目标读者确实不了解）
2. **强化关键句**：每个段落的第一句必须是核心句，用**加粗**标记
3. **可视化升级**：
   - 将文字描述转化为表格（对比、流程、层次）
   - 用文字描述图表结构（如"核心论点→三维度→九要素"的树状图）
   - 关键公式用代码块或LaTeX风格呈现

#### Step 3.3: 执行修改与验证

按照诊断报告的优先级，逐条应用升级策略：

1. **先处理高严重度问题**（🔴）
2. **批量处理同类问题**（如所有"论据不足"的点一起补充）
3. **每轮修改后朗读全文**，检查流畅度和一致性
4. **最终验证**：对照初稿→终稿升级检查清单（见附录A）

**输出**：终稿 + 修改说明（记录主要改动和原因）

### Phase 4: 输出与归档

1. **格式化输出**：根据用途选择格式（Markdown/Word/飞书文档等）
2. **元信息归档**：记录写作背景、核心论点、主要证据来源、修改要点
3. **技能沉淀**：将本次写作中形成的新洞察、新方法，更新到本 SKILL.md 的"实践案例"部分

## 附录A：初稿→终稿升级检查清单

### 结构层

- [ ] 核心论点是否一句话能说清？
- [ ] 核心论点是否前置（前10%篇幅内）？
- [ ] 章节顺序是否符合逻辑（时间/重要性/复杂度递进）？
- [ ] 每章是否有明确的子论点，且与核心论点关联？
- [ ] 是否存在信息重复？如有，合并或删除。

### 内容层

- [ ] 每个核心论点是否有≥3个证据支撑（数据/案例/引用）？
- [ ] 证据是否具体（有数字、有细节、有来源）而非笼统描述？
- [ ] 是否形成了可命名的概念/理论（XXX理论/模型/框架）？
- [ ] 是否有可视化呈现（表格/图表描述/公式）？
- [ ] 是否考虑了反面观点和边界条件？

### 表达层

- [ ] 删减20-30%冗余内容（重复论证、过度铺垫、已知常识）
- [ ] 每个段落第一句是否为核心句且加粗？
- [ ] 专业术语使用是否一致（首次出现是否解释）？
- [ ] 数字和单位是否统一（万/亿，美元/人民币）？
- [ ] 朗读全文，检查流畅度和节奏感

### 批判性思维层

- [ ] 是否明确说明了前提假设？
- [ ] 是否讨论了相关性≠因果性的风险？
- [ ] 是否界定了理论的适用范围和边界条件？
- [ ] 是否考虑了能力外包和人类技能退化的风险？
- [ ] 是否回应了最聪明的反对者可能的反驳？

## 附录B：实践案例（持续更新）

### 案例1：《Token竞争力理论》写作过程

**背景**：基于 OpenClaw 现象的深度分析文章

**四层诊断结果**：
- 类型：分析型（现象→机制→趋势）
- 读者：专业同行（AI/科技领域从业者）
- 核心论点：Token消耗量正在成为脑力劳动竞争力的核心度量
- 证据状态：已有部分素材（新闻报道、用户反馈），需补充学术支撑

**初稿问题诊断**：
- 🔴 核心论点前置不足（在引言后才出现）
- 🔴 理论深度不够（停留在现象描述）
- 🟡 批判性反思薄弱（只谈了边际收益递减）
- 🟢 表达基本清晰

**升级策略应用**：
1. **倒金字塔结构**：增加"零、核心论点"章节，前置核心洞察+前提假设
2. **理论品牌化**：将"Token消耗"升级为"Token竞争力理论"，公式化表达
3. **批判性深化**：增加"能力外包"讨论，承认"幻觉问题"局限

**终稿亮点**：
- 核心论点一句话可概括
- 形成可命名的"Token竞争力理论"
- 包含可视化图表（公式图、人力资本折旧图）
- 批判性反思独立成章，回应5个关键问题

**技能沉淀**（已更新到本 SKILL.md）：
- 倒金字塔结构适用于决策者或时间稀缺读者
- 理论品牌化需要：命名+公式化+可视化三要素
- 批判性反思必须包含：前提假设、边界条件、能力外包风险
