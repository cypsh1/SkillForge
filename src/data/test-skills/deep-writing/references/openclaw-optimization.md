# OpenClaw 专用写作优化指南

## OpenClaw 核心优势在写作中的应用

### 1. 长上下文利用策略

**优势**：OpenClaw 支持超长上下文（20万+ Token），可一次性载入大量参考资料。

**写作应用场景**：

1. **完整载入参考资料**
   - 将所有相关文献、报告、笔记一次性载入上下文
   - 避免反复切换窗口查找资料
   - 利用 OpenClaw 的记忆能力，建立跨文档关联

2. **保持写作连贯性**
   - 将已写内容始终保持在上下文中
   - 随时引用前文，确保逻辑一致性
   - 支持"回到第X章修改"而不丢失上下文

3. **多文档并行处理**
   - 同时参考多个来源，进行交叉验证
   - 对比不同观点，形成平衡论证
   - 提取共性和差异，形成洞察

**操作技巧**：

```markdown
## 上下文管理指令

1. 载入参考资料：
   "请将 [文件路径/URL] 的内容完整载入上下文，我需要参考其中的 [具体内容]"

2. 回顾前文：
   "请回顾第X章的核心观点，确保本章论证与之呼应"

3. 建立关联：
   "将[前文概念A]与[当前概念B]进行对比，分析它们的关联和差异"

4. 保持连贯：
   "在继续写作前，请总结当前文档的核心论点和已完成的章节结构"
```

### 2. Agent 模式分块策略

**优势**：OpenClaw 的 Agent 模式可将复杂任务拆分为子任务，并行处理。

**写作应用场景**：

1. **并行章节撰写**
   - 将文章拆分为多个相对独立的章节
   - 多个 Agent 分别撰写不同章节
   - 最后由主 Agent 统一整合和润色

2. **专项任务外包**
   - 数据分析 Agent：处理数据、生成图表描述
   - 案例研究 Agent：搜集和整理案例
   - 文献综述 Agent：整理和总结文献
   - 校对 Agent：检查语法、逻辑、一致性

3. **多版本并行**
   - 同一章节由不同 Agent 撰写多个版本
   - 对比各版本优劣，融合最佳元素
   - 或根据场景选择最适合的版本

**操作技巧**：

```markdown
## Agent 分块指令

1. 创建专项 Agent：
   "创建一个 [数据分析/案例研究/文献综述] Agent，负责 [具体任务]"

2. 并行章节撰写：
   "将本文拆分为 [章节A]、[章节B]、[章节C]，分别创建 Agent 并行撰写"

3. 任务结果整合：
   "请整合各 Agent 的输出，确保：
   - 逻辑连贯性
   - 术语一致性
   - 风格统一性"

4. 质量检查：
   "创建校对 Agent，检查全文：
   - 语法和拼写错误
   - 逻辑漏洞和跳跃
   - 前后不一致之处
   - 表达冗余之处"
```

### 3. 多轮迭代策略

**优势**：OpenClaw 支持持续的上下文保持，可进行多轮深度迭代。

**写作应用场景**：

1. **分层递进式迭代**
   - 第一轮：快速产出完整框架和初稿（不求完美，求完整）
   - 第二轮：逐段深化（补充证据、强化论证、优化表达）
   - 第三轮：整体润色（一致性检查、节奏调整、开头结尾强化）

2. **问题导向式迭代**
   - 识别初稿中的具体问题（论点不清、证据不足、逻辑跳跃等）
   - 每轮聚焦解决一个或一类问题
   - 保持修改记录，便于回溯和比较

3. **反馈驱动式迭代**
   - 生成特定版本的初稿
   - 获取反馈（自我审读、他人评阅、AI辅助分析）
   - 根据反馈进行针对性修改
   - 重复直到满足发布标准

**操作技巧**：

```markdown
## 多轮迭代指令

1. 第一轮（快速初稿）：
   "请在 [时间限制] 内产出完整初稿，要求：
   - 覆盖所有章节
   - 不求完美，但求完整
   - 标记需要后续深化的部分"

2. 第二轮（逐段深化）：
   "针对 [具体章节]，进行深化：
   - 补充具体证据和数据
   - 强化逻辑论证
   - 优化表达精炼度"

3. 第三轮（整体润色）：
   "进行全文润色，确保：
   - 术语一致性
   - 逻辑连贯性
   - 开头吸引人，结尾有力
   - 节奏流畅，无冗余"

4. 问题定位：
   "请识别初稿中的 [高/中/低] 严重程度问题，分类整理，
   并建议每轮迭代重点解决哪类问题"

5. 版本比较：
   "请比较 [版本A] 和 [版本B]，分析：
   - 主要改进点
   - 仍存在的不足
   - 下一步优化建议"
```

## OpenClaw 写作专用指令集

### 快速启动指令

```markdown
"启动深度写作模式，执行四层诊断：
1. 写作类型：[用户提供或AI辅助选择]
2. 目标读者：[用户提供或AI辅助选择]
3. 核心论点：[用户提供或AI生成候选]
4. 证据状态：[用户确认]"
```

### 结构规划指令

```markdown
"基于四层诊断结果，生成文章结构：
- 选择最适合的结构类型（倒金字塔/逻辑递进/时间线/步骤/辩证）
- 列出各章节标题和核心信息点
- 预估各部分字数分配
- 标注需要重点深化的章节"
```

### 初稿生成指令

```markdown
"使用OpenClaw优化模式生成初稿：
- 一次性载入所有参考资料到上下文
- 启用Agent模式，复杂章节创建子任务
- 每完成一章，简要总结并确认方向
- 标记需要后续深化的部分"
```

### 修改完善指令

```markdown
"执行初稿→终稿升级：
1. 诊断初稿问题（结构/内容/表达/批判性）
2. 选择升级策略（倒金字塔/理论化/批判深化/表达精炼）
3. 执行多轮迭代修改
4. 最终验证（附录A检查清单）"
```

### 问题记录指令

```markdown
"记录本次写作过程中的问题：
- 在哪个阶段卡住了？
- AI生成的内容有什么不满意？
- 实际输出与预期的差距在哪里？
- 有什么方法或技巧在这次写作中特别有效？

将记录添加到改进日志，用于Skill优化。"
```
