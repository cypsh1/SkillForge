# Deep Writing 快速参考卡片

## 一、四层诊断速查表

### 第一层：写作类型（选择最接近的）

| 类型 | 核心特征 | 关键问句 |
|------|----------|----------|
| 分析型 | 现象→机制→趋势 | "为什么会这样？" |
| 论证型 | 论点→论证→回应 | "这个观点成立吗？" |
| 叙事型 | 背景→冲突→启示 | "发生了什么？有什么意义？" |
| 指导型 | 问题→步骤→注意事项 | "怎么做？要注意什么？" |
| 思辨型 | 概念→推演→边界 | "这意味着什么？边界在哪里？" |

**决策辅助**：
- 如果主要目的是"解释现象并揭示机制"→分析型
- 如果主要目的是"论证观点并回应质疑"→论证型
- 如果主要目的是"讲述故事并提取启示"→叙事型
- 如果主要目的是"指导操作并规避风险"→指导型
- 如果主要目的是"探索概念并推演边界"→思辨型

### 第二层：目标读者（选择主要的1-2个）

| 读者类型 | 核心需求 | 写作策略关键词 |
|----------|----------|----------------|
| 决策者 | 快速把握结论和ROI | 倒金字塔、Executive Summary、可扫描 |
| 专业同行 | 方法论和深度洞察 | 理论深度、证据充分、承认局限 |
| 跨界学习者 | 理解概念和上下文 | 概念解释、类比丰富、循序渐进 |
| 公众/大众 | 相关性和情感共鸣 | 故事驱动、案例具象、节奏流畅 |
| 实践执行者 | 可操作步骤和避坑 | SOP清晰、工具推荐、检查点 |

### 第三层：核心论点（AI辅助生成候选）

**操作流程**：

1. **用户提供明确论点**：直接确认并提炼
2. **用户提供模糊方向**：AI生成3-5个候选论点供选择
3. **用户无明确想法**：AI基于主题和类型，生成候选论点

**候选论点生成模板**：

```
基于您的主题「[主题]」，以下是[数量]个候选核心论点：

1. 【[类型，如：范式转移型/效率革命型/风险警示型]】[核心论点]
   → 适用场景：[什么情况下这个观点最有说服力？]
   → 关键证据：[需要什么数据/案例支撑？]

2. 【[类型]】[核心论点]
   → 适用场景：
   → 关键证据：

[继续...]

请选择最接近您观点的选项，或提出您的核心论点。
```

### 第四层：证据状态

| 状态 | 特征 | 后续策略 |
|------|------|----------|
| **已有充分素材** | 数据、案例、引用已齐备 | 直接进入结构规划和写作 |
| **有部分素材** | 核心数据/案例有，但需补充 | 写作过程中标记需补充点，并行搜集 |
| **素材稀缺** | 观点先行，证据不足 | 先进行深度调研阶段，再进入写作 |
| **需要生成性研究** | 需要访谈、实验、数据收集 | 设计研究方案，写作延后 |

## 二、结构决策速查表

### 结构类型选择

| 结构类型 | 适用场景 | 核心特征 | 关键词 |
|----------|----------|----------|--------|
| **倒金字塔** | 决策者、时间稀缺读者 | 核心信息 → 关键细节 → 背景 | 前置、可扫描 |
| **逻辑递进** | 分析型、论证型文章 | 问题 → 分析 → 论证 → 结论 | 层层深入 |
| **时间线/叙事** | 叙事型、案例复盘 | 背景 → 冲突 → 高潮 → 结局 | 故事张力 |
| **步骤/模块化** | 指导型、操作手册 | 步骤1 → 步骤2 → ... → 注意事项 | 可操作性 |
| **辩证/对话** | 思辨型、伦理讨论 | 正论 → 反论 → 综合 → 新问题 | 思维张力 |

### 结构规划输出模板

```markdown
## 文章结构大纲

### 核心信息
- **核心论点**：[一句话概括]
- **目标读者**：[读者类型]
- **预期字数**：[总字数，各章节分配]

### 章节结构

#### [章节序号]. [章节标题]
- **核心信息**：[本章要传达的核心信息]
- **字数预估**：[X字]
- **关键信息点**：
  1. [信息点1]
  2. [信息点2]
  3. [信息点3]
- **需重点深化**：是/否，如果是，说明深化方向

[重复上述结构...]

### 整体节奏
- **开头策略**：[如何吸引读者？]
- **高潮位置**：[核心洞察放在哪里？]
- **结尾策略**：[如何收尾有力？]

### 待解决问题
1. [需要进一步研究的问题]
2. [需要用户确认的问题]
```

## 四、修改完善速查表

### 初稿→终稿升级策略选择

| 初稿问题 | 适用升级策略 | 关键操作 |
|----------|--------------|----------|
| 核心论点不清晰/前置不足 | **A. 倒金字塔强化** | 增加"零章"或Executive Summary，核心论点前置 |
| 理论深度不够/概念描述层面 | **B. 理论品牌化** | 概念→命名+公式化+可视化，形成理论模型 |
| 缺乏批判性/单向度论证 | **C. 批判性深化** | 增加前提假设、边界条件、能力外包风险讨论 |
| 表达冗长/结构混乱 | **D. 表达精炼** | 删减20-30%，强化关键句，增加表格/图表 |
| 多个问题并存 | **组合策略** | 按优先级组合A/B/C/D |

### 策略 A: 倒金字塔强化

```markdown
## 倒金字塔强化操作指南

### 适用信号
- 核心论点出现在文章前30%之后
- 开头铺垫过长（超过全文10%）
- 读者需要读完全文才能抓住重点

### 操作步骤

#### Step 1: 提炼核心论点
用一句话回答："如果读者只能记住一句话，应该是什么？"

[在此处写下核心论点]

#### Step 2: 提炼3个关键洞察
支撑核心论点的3个最重要子观点或数据。

1. [洞察1]
2. [洞察2]
3. [洞察3]

#### Step 3: 重构开头

在文章最前端增加以下结构（建议300-500字）：

---

**【零、核心论点】**（或 **【Executive Summary】**）

[核心论点]

**三个关键洞察**：
1. [洞察1的简洁表述]
2. [洞察2的简洁表述]
3. [洞察3的简洁表述]

**本文结构**：[用1-2句话概括文章如何展开]

---

#### Step 4: 调整正文
- 将原开头中必要的背景信息，压缩后移至第一章或分散到各章
- 确保各章与"零章"的核心论点和关键洞察呼应
- 删除与核心论点无关的铺垫性内容

### 检查清单

- [ ] 核心论点一句话能说清
- [ ] 核心论点出现在文章前5%位置
- [ ] 3个关键洞察具体、可验证
- [ ] 背景信息未过度前置
- [ ] 各章与核心论点形成呼应
