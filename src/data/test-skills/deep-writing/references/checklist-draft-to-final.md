# 初稿→终稿升级检查清单

## 结构层检查

- [ ] 核心论点是否一句话能说清？
- [ ] 核心论点是否前置（前10%篇幅内）？
- [ ] 章节顺序是否符合逻辑（时间/重要性/复杂度递进）？
- [ ] 每章是否有明确的子论点，且与核心论点关联？
- [ ] 是否存在信息重复？如有，合并或删除。

## 内容层检查

- [ ] 每个核心论点是否有≥3个证据支撑（数据/案例/引用）？
- [ ] 证据是否具体（有数字、有细节、有来源）而非笼统描述？
- [ ] 是否形成了可命名的概念/理论（XXX理论/模型/框架）？
- [ ] 是否有可视化呈现（表格/图表描述/公式）？
- [ ] 是否考虑了反面观点和边界条件？

## 表达层检查

- [ ] 删减20-30%冗余内容（重复论证、过度铺垫、已知常识）
- [ ] 每个段落第一句是否为核心句且加粗？
- [ ] 专业术语使用是否一致（首次出现是否解释）？
- [ ] 数字和单位是否统一（万/亿，美元/人民币）？
- [ ] 朗读全文，检查流畅度和节奏感

## 批判性思维层检查

- [ ] 是否明确说明了前提假设？
- [ ] 是否讨论了相关性≠因果性的风险？
- [ ] 是否界定了理论的适用范围和边界条件？
- [ ] 是否考虑了能力外包和人类技能退化的风险？
- [ ] 是否回应了最聪明的反对者可能的反驳？

---

## 评分标准

每项检查完成后，可以按以下标准评分：

| 评分 | 含义 | 行动 |
|------|------|------|
| ✅ 优秀 | 完全满足要求，无需改进 | 保持 |
| ⚠️ 良好 | 基本满足，有小瑕疵 | 微调 |
| ❌ 需改进 | 不满足要求，需要修改 | 重点改进 |

---

## 使用建议

1. **初稿完成后**：先进行自我检查，标记需要改进的项目
2. **修改过程中**：每完成一轮修改，重新检查相关项目
3. **终稿确认前**：完整检查一遍，确保所有关键项目都达到良好以上
4. **发布后复盘**：回顾检查清单，记录哪些项目最容易出问题，下次重点注意
