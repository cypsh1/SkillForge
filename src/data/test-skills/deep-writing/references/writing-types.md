# 写作类型指南

## 分析型

**特征**：剖析现象 → 揭示机制 → 推导趋势

**适用场景**：行业分析、竞争研究、技术解读、政策评估

**结构模板**：
1. 现象描述（发生了什么？）
2. 机制分析（为什么会这样？）
3. 趋势推导（接下来会怎样？）
4. 启示/建议（我们该怎么办？）

**核心技巧**：
- 使用"五问法"（连续追问5个为什么）深挖机制
- 区分 proximate cause（近因）和 ultimate cause（远因）
- 趋势预测要明确时间窗口和触发条件

---

## 论证型

**特征**：提出论点 → 多维论证 → 回应质疑

**适用场景**：观点文章、政策建议、战略提案、学术论证

**结构模板**：
1. 论点陈述（核心观点一句话）
2. 论证展开（2-4个维度，每个维度有证据支撑）
3. 反方回应（预判主要质疑，逐一回应）
4. 结论升华（从具体到抽象，从当下到长远）

**核心技巧**：
- 使用"如果...那么..."句式明确因果链
- 每个论证维度要有独立且充分的证据
- 回应质疑时先承认对方合理之处，再指出局限

---

## 叙事型

**特征**：背景铺陈 → 冲突展开 → 结局/启示

**适用场景**：案例复盘、创业故事、变革记录、人物特稿

**结构模板**：
1. 背景设定（时间、地点、人物、常态）
2. 冲突出现（什么打破了常态？）
3. 发展/高潮（如何应对？结果如何？）
4. 结局/启示（学到了什么？有什么普遍意义？）

**核心技巧**：
- 使用具体细节（感官描写、对话、动作）而非概括陈述
- 冲突要有"意外性"和"代价"，才能产生张力
- 启示要从具体故事上升到普遍规律，但避免过度引申

---

## 指导型

**特征**：问题界定 → 步骤拆解 → 注意事项

**适用场景**：操作手册、实施指南、方法论、教程

**结构模板**：
1. 目标/适用范围（解决什么问题？适合谁用？）
2. 前置条件（需要什么基础？准备什么资源？）
3. 步骤详解（每一步的输入、操作、输出）
4. 常见问题/注意事项（什么容易出错？如何应对？）

**核心技巧**：
- 每一步要有明确可验证的输出，便于确认是否正确执行
- 使用编号列表而非段落，便于快速定位和回溯
- 注意事项要具体到"如果发生X，则做Y"，而非泛泛提醒

---

## 思辨型

**特征**：概念辨析 → 逻辑推演 → 边界探索

**适用场景**：哲学讨论、范式批判、未来构想、伦理辩论

**结构模板**：
1. 概念界定（核心术语的定义、范围、历史演变）
2. 正反论证（支持观点A的理由 vs 支持观点B的理由）
3. 综合/超越（如何超越二元对立？新的框架是什么？）
4. 边界/问题（新框架的适用边界？未解决的问题？）

**核心技巧**：
- 概念界定要区分"日常用法"、"学术定义"和"本文特指"
- 正反论证时要找到最强版本的对方观点（"steel man"），而非稻草人
- 综合阶段要有创造性，不只是"各打五十大板"，而是提出新的概念框架
